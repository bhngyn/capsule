# Capsule クイックスタート

*ウェブを、証拠とともに保存する。五分で始めましょう。*

Capsule は、調査担当者がウェブページや動画、投稿を、後から検証に耐える形で保存するためのツールです。一回の保存ごとに、ページをそのままの姿で記録し、メディアがあれば取得し、結果に署名します。受け取った相手が、後から内容に変更がないことを確認できます。

このガイドは、最初の保存までを案内します。

---

## 用意するもの

- **macOS 12 以降**または **Windows 10/11** が動作するパソコン。
- **Docker Desktop**（無料）。Capsule はこの中で動くので、Python やブラウザ、yt-dlp を自分で入れる必要はありません。
- 最初のダウンロードに **2 GB ほど** のディスクの空き容量。

> Docker Desktop は無料のツールで、これがあれば Capsule はパソコンで動きます。<https://www.docker.com/products/docker-desktop> から入手できます。

---

## ワンクリックで起動

1. Docker Desktop を入れて、一度だけ開いてください。メニューバー（macOS）またはタスクトレイ（Windows）にアイコンが表示され、稼働中であることを確認します。
2. Capsule フォルダ内のランチャーを **ダブルクリック** します。
    - macOS: `Capsule.command`
    - Windows: `Capsule.bat`
3. 初回はランチャーが Capsule（およそ 2 GB）をダウンロードします。二回目以降はおよそ三秒で立ち上がります。
4. ブラウザが開き、保存画面が表示されます。

これで準備は完了です。ターミナルもコマンドも使いません。

![保存画面](screenshots/dashboard.en.png)

## はじめての保存

1. URL 欄にリンクを貼り付けます。動画でも、ツイートでも、ニュース記事でも構いません。
2. **保存** を押します。
3. Capsule は順番に四つの工程を実行します。ページのスナップショット、メディアの取得、すべてのファイルのハッシュ計算、そして結果への署名です。各工程が点灯して進みます。
4. 完了すると、保存したアイテムが「最近の保存」に緑色の整合性バッジ付きで現れます。

リンク一つにつき、Capsule が保存するもの。

- ページ全体のスクリーンショット
- ページの自己完結型スナップショット（MHTML）
- ページとそのサブリソースをまとめた WARC アーカイブ
- 動画や音声がある場合はそのファイル
- 技術的な詳細を記録した JSON のサイドカー
- すべてのファイルの MD5 と SHA-256 ハッシュ
- 自分も他者も検証できる署名

メディアが見つからないページでも、ページのスナップショットは残ります。手元には必ず何かが残る、というわけです。

---

## ファイルの保存先

Capsule は、ふだん使うフォルダと同じように開ける場所に保存します。

- macOS: `~/Documents/Capsule/`
- Windows: `%USERPROFILE%\Documents\Capsule\`

ケースごとに専用のサブフォルダができます。さらにその下では、保存ごとに一つのアイテムフォルダが作られ、メディアファイル、ページのスナップショット、マニフェスト PDF、署名済みのサイドカーが、すべて同じ場所にまとまります。誰かに一件分だけ渡したいときは、そのアイテムフォルダを丸ごとコピーすれば足ります。

---

## 停止と再起動

- **停止する場合**：Docker Desktop を終了するか、ターミナルで `docker stop capsule` を実行します。
- **再び起動する場合**：ランチャーをダブルクリックします。
- パソコンの電源を入れたとき、Capsule は **自動では立ち上がりません**。使いたいときだけ立ち上げ、それ以外のときは邪魔になりません。

---

## 次にできること

- ダウンローダーが第 1 版の画面のすべてです。リンクを貼り付け、四工程の進行を見て、結果を「最近の保存」で確認します。
- ヘッダーの歯車から **設定** を開けます。言語の切り替え、署名鍵のフィンガープリントの確認、拡張機能のペアリング、yt-dlp の更新の確認ができます。
- 証拠のためのデータ、つまりケースごとに整理されたフォルダ、ハッシュで連鎖した監査ログ、署名された証拠エクスポートのバンドルは、保存ごとに作られ続けます。`~/Documents/Capsule/quick-captures/` のディスク上、および API から参照できます。証拠の引き渡しと検証については **ユーザーガイド** をご覧ください。
- Docker 自体に関する問題は、Docker Desktop の **Help** メニューが助けになります。

何か問題が起きた場合、どのエラーにも「技術的な詳細を表示」のボタンが付いています。その内容をバグ報告に貼り付けてお送りいただければ、サポートにつながります。
